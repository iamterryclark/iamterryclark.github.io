Aim

The main idea for this project was to create a personal website, where I could promote my skills and display the projects which I am, and will be, most proud of during my time at Goldsmiths.

I wanted to develop a clean and modern looking site which took advantage of web design trends between 2014-15. This includes a large header image, a one page flat layout, clean and crisp fonts and most of all responsive.

After previously having read lots of resources on UXPin (UXpin, 2010) I was able to find lots of great examples of websites, one of which caught my eye the most (Weainplastic.com - Roland Lösslein. 2015) as it had some very interesting animated SVG graphics and fading/navigtion animations.

To achieve this I firstly created a mockup and then developed the site so that could spend more time on the content and have an understanding on the style of the site beforehand. Please see image attached here

Code/Information I used directly

The fonts on the site came from Google fonts (Lato), I imported these through the CSS as I felt this make the html much cleaner.

I have also used font-awesome (Font-Awesome, 2015) a library of icons which are free to use, which help to indicate certain features of the site such as a down arrow which signifies that there is some more content below. Furthermore the hamburger menu is a commonly used icon although I have started to read more about its issues and may look at other choices of navigation in the future.

revealOnScroll.js has given me the recursive fading effects when the user scrolls up and down the site. I feel this attracts the user's attention to the important parts of my site and adds an element of fun/interaction to it. I used a tutorial that I found online to implement it but there still seem to be issues with this and the smooth-scolling effects I have. (http://blog.webbb.be/, 2014)

animate.css controls which kind of animation I would like to display on the site, it contains a library of different effect which can be see (Animate.css - Daniel Eden, 2015)

modernizr.js helps with mobile interactivity and was needed to run revealOnScroll.js

From my previous knowledge in website building, I understood that I need to use viewport in order to display the site correctly on mobile and not like a miniature version of my full desktop site. (W3C - Responsive Web Design, 1999)

Part of the extra criteria was to account for the changes in HTML5 and adapt this to work in IE8 and thus I found this resource and included it into the index.php file (W3C - HTML5 Browser Support, 1999)

Code I Adapted

I borrowed the code of the navigation (Weainplastic.com - Roland Lösslein. 2015) which included the transition of the top of the navigation and needed then to further adapt this for the mobile navigation by using the same effects but having the menu slide in from the left.

I also used a piece of code which I found through code pen, an online code editor for prototyping, to help with the animation of the closing navigation button when in the mobile version of the site. I needed to adapt this by changing the size of the span widths and correcting the spacing between them to match with my font-awesome hamburger icon. (CodePen - Jesse Couch. 2015)

For the contact form I used a video which I had download which gave full details on how to use PHP mail() function to send myself an email via a contact form. I adjusted the text and also styled the form to match the style of the site. (Udemy - Rob Percival, 2014)

Things I Like

I am really happy with the layout of the site, I believe it has a consistant style to it including colours and fonts used along with a good overall placement of content.

The complexity is also satisfying to me as I wanted to push my skills with PHP, animations and building a compelete website from scratch.

Current Challenges

There are some current issues with the site which I was unable to fix and would like some further feedback regarding this, if possible.

Unfortunately I was unable to implement the Jquery accordion / Flexslider into the site as there seems to be in clash between the code and this stops the slider from displaying. This was going to display some of the websites I have already worked on

Another more visible issue is that the smooth-scrolling and revealOnScroll.js clash and do not work together. The navigation opens but the links do not scroll to the specific sections. This is due to html and body CSS properties being set to a height: 100%; and this fixes the animation effects but stops the smooth scroll from working. Futhermore, I found that if I changed the height: auto; and set a min-height: 100%; then this resolves the smooth scroll issue but the animation will only work on load and not repeatedly as the user scrolls up and down the page.

The last page which I created was the visuals page where I could display my favourite Processing sketches. Unfortunately one sketch does not load properly and I cannot find the reason why. I have tried both in-line (code pasted into the html document) and importing methods (using data-processing-target="" within the canvas element.

Lastly, I would like to create a HTML CV page rather than linking to CV but due to time restrictionsI was unable to complete this for the project but will be something I later implement.

Evaluation

This site has been verified through the W3C Validation checker and now comes back with no errors, I have further checked the CSS and tried to find a JavaScript's validation tool. Errors appear on font-awesome.css but with reference to the site 3 this shows that this is a normal occurence.
